Mysql Cluster 集群搭建

Mysql集群方案的特点是无需共享存储，其数据通过主从同步来实现各节点的一致性。整个集群方案价格如下图，其中共有3类节点。

安装包及实验环境

虚拟机：Virtual Box 5.0
操作系统：CentOS 7.0
虚拟机1：192.168.59.104 （三种节点均运行一个实例）
虚拟机2：192.168.59.105 （三种节点均运行一个实例）

下载 MySQL-Cluster-gpl-7.4.9-1.el7.x86_64.rpm-bundle.tar

解压后在所有机器上安装以下两个rpm包：
MySQL-Cluster-client-gpl-7.4.9-1.el7.x86_64.rpm
MySQL-Cluster-server-gpl-7.4.9-1.el7.x86_64.rpm

3类节点的配置和启动顺序为: 管理节点 -> 数据节点 -> SQL服务节点

管理节点

管理节点定义了集群的配置信息，如各类节点数目，节点ID，节点地址等。集群启动阶段各个数据和SQL节点都需要连接管理节点，获取配置信息。集群运行时通过这个该节点查看及管理集群。

配置文件

1.> more /var/lib/mysql-cluster/config.ini
2.[ndb_mgmd]      # 定义管理节点
3.nodeid=1
4.hostname=192.168.59.104
5.datadir=/var/lib/mysql-cluster/
6.
7.[ndb_mgmd]
8.nodeid=2
9.hostname=192.168.59.105
10.datadir=/var/lib/mysql-cluster/
11.
12.[ndbd default]  # ndbd的默认配置信息，数据节点的通用配置写在此处
13.NoOfReplicas=2  # NoOfReplicas表示数据份数，如果为1，会有数据节点单点故障
14.DataMemory=80M  # 数据节点使用的共享内存，生产环境中应根据数据量的大小计算出来。
15.IndexMemory=18M
16.
17.[ndbd]          #定义数据节点
18.nodeid=11
19.hostname=192.168.59.104
20.datadir=/var/lib/mysql/data
21.
22.[ndbd]
23.nodeid=12
24.hostname=192.168.59.105
25.datadir=/var/lib/mysql/data
26.
27.[mysqld]        #定义SQL服务节点
28.id=21
29.hostname=192.168.59.104
30.
31.[mysqld]
32.nodeid=22
33.hostname=192.168.59.105
34.
35.[mysqld]
36.nodeid=23       #没有定义节点地址的时候，表示这个节点可能来自任何地址。

配置及启动

1.> mkdir -p /var/lib/mysql-cluster/config.ini
2.> vim /var/lib/mysql-cluster/config.ini
3.> chown -R mysql:mysql /var/lib/mysql-cluster/
4.> ndb_mgmd -f /var/lib/mysql-cluster/config.ini --initial #--initial参数为第一次启动时使用

如果有多个管理节点，第一次启动时需要把所有管理节点都启动，否则运行ndb_mgm时会报错。

查看集群状态

1.> ndb_mgm -e show
2.Connected to Management Server at: 192.168.59.104:1186
3.Cluster Configuration
4.---------------------
5.[ndbd(NDB)]     2 node(s)
6.id=11 (not connected, accepting connect from 192.168.59.104)
7.id=12 (not connected, accepting connect from 192.168.59.105)
8.
9.[ndb_mgmd(MGM)] 2 node(s)
10.id=1    @192.168.59.104  (mysql-5.6.28 ndb-7.4.9)
11.id=2 (not connected, accepting connect from 192.168.59.105)
12.
13.[mysqld(API)]   3 node(s)
14.id=21 (not connected, accepting connect from 192.168.59.104)
15.id=22 (not connected, accepting connect from 192.168.59.105)
16.id=23 (not connected, accepting connect from any host)

上图中，由于在管理节点中定义了节点地址，因此nbd_mgmd进程正在等待192.168.59.104上的节点11，21和192.168.59.105上的节点12，22。而没有定义节点地址的SQL服务节点23可以来自任何机器。

数据节点

配置文件

数据节点的配置信息检索顺序如下：

# ndbd –help
…
Default options are read from the following files in the given order:
/etc/my.cnf /etc/mysql/my.cnf /usr/etc/my.cnf ~/.my.cnf
The following groups are read: mysql_cluster ndbd
…

1.> cat /etc/my.cnf
2....
3.[mysql_cluster]
4.ndb-connectstring=192.168.59.104,192.168.59.105 # 定义管理节点连接信息
5....

其余配置信息从管理节点中获取。

启动命令

1.> ndbd --initial  # --initial参数为第一次启动时使用

SQL服务节点

配置文件：

# mysqld –verbose –help
…
Default options are read from the following files in the given order:
/etc/my.cnf /etc/mysql/my.cnf /usr/etc/my.cnf ~/.my.cnf
The following groups are read: mysql_cluster mysqld server mysqld-5.6
…

1.> cat /etc/my.cnf
2....
3.[mysqld]
4.ndbcluster
5.datadir=/var/lib/mysql/data
6.socket=/tmp/mysql.sock
7.port=3307
8.ndb-connectstring=192.168.59.104,192.168.59.105 # 定义管理节点连接信息
9....

启动命令

1.> /usr/share/mysql/mysql.server start

所有节点启动完毕后，再检查集群状态：
